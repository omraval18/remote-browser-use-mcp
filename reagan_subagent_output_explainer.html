Subagent Output Rendering

This is the current TUI model in crates/browser-use-tui/src/transcript.rs and render.rs. Subagent output is not one raw stream. It is summarized twice: once for the parent workbench activity projection, and once for the parent transcript UI.

1. Event Flow

Parent session events

session.input user prompt
agent.spawned child id + nickname
agent.wait.started optional waiting target
agent.completed short payload result
→

Child session events

session.input child task
file.read / file.list / command.started
model.stream_delta child draft text
session.done full child result
→

Parent visible transcript

active: live child activity summary
committed: child recent actions + finished + short result
not shown: full child prompt/result as parent output

2. What Each Subagent Block Means

agent.spawned
Used to name the child, but intentionally not rendered as its own transcript row. The code comment says rendering it separately made duplicate subagent blocks.
Active child block
When a child session is active, active_child_session() returns it and the parent active viewport becomes : subagent {nickname}.
working
Always included as the first active line for a live child.
waiting on ...
Included only when the current parent turn has agent.wait.started. It comes from wait_agent_started_label().
Recent child actions
Built from the child session events by recent_child_activity_lines(). Examples: read README.md, list ., search "query", run cargo test, artifact path.
Child streaming text
Usually hidden in the TUI child summary. child_activity_line() returns None for model.stream_delta and model.thinking_delta. Protocol projection may mention streaming in generic activity, but the transcript UI does not show the raw draft.
agent.completed
Creates a committed : subagent {nickname} timeline node. It includes up to 4 recent child activity lines, then finished, then up to 3 preview lines from payload.result.
Full child result
Deliberately not replayed into the parent transcript. Tests assert strings from child session.done do not appear at top level unless the parent includes a short result in agent.completed.payload.result.

In native scrollback mode, render_main() renders only the active viewport in the Ratatui body. The already-committed transcript is above it in native terminal scrollback.

The layout then calls main_layout_areas(..., attach_bottom_to_body = true). In this mode the body is placed at the top of the current Ratatui frame, the composer is placed directly under it, and the remaining blank space goes below the composer.

That was meant to keep the composer attached to active output. But visually, when the native scrollback above the frame is also visible in the terminal, the small active subagent block can look like it lands in the middle of the total viewport instead of growing from the bottom.

4. The Two Rendering Paths

Native scrollback active

render_main() body = active_viewport_lines_with_stream_skip(...) attach_bottom_to_body = native_scrollback_active && !body.is_empty() main_layout_areas(...) Result: committed transcript = terminal scrollback active subagent block = Ratatui body composer = Ratatui bottom pane

Native scrollback inactive

work_lines() lines = all_scrollback_lines(model) if running: lines += active_viewport_lines(model) Result: committed transcript + active block are one Ratatui body list then visible_main_body_lines crops it

5. Diagnosis

The subagent content itself is a compact summary, not raw child output. The placement issue is layout-level: in native scrollback mode the active body is small, the composer attaches immediately below it, and the spacer sits below the composer. That can make the active subagent block feel centered or floating instead of bottom-aligned with the composer at the terminal bottom.

If the desired convention is “output grows downward until the composer reaches the bottom, then old lines scroll upward,” the subagent active block should follow the same visual-space rule as other live output: active output plus composer should be bottom-docked when there is spare frame height, while committed output remains native scrollback.
