Rendering Logic, In HTML Terms

native scrollback Ratatui inline viewport committed vs active why the jump happens

This page treats the terminal app like a web page so the layout problem is visible. The real app is not HTML: the old transcript is printed into terminal-native scrollback, while the live region and composer are drawn by Ratatui inside an inline viewport.

The Current Shape

The app behaves like this pseudo DOM. The important part is that the first block is owned by the terminal, not by Ratatui.

Pseudo HTML
<body class="terminal">
  <div id="native-scrollback">
    <!-- committed / inactive transcript -->
    > Find the top 5 Hacker News posts
    Top 5 Hacker News posts...
    source https://news.ycombinator.com
  </div>

  <div id="ratatui-inline-viewport">
    <div id="active-live-area">
      <!-- thinking / streaming / running tool -->
    </div>

    <div id="composer">
      Type to steer the agent...
    </div>
  </div>
</body>
Ownership Split
Committed transcript
  Printed into terminal-native scrollback.
  It is append-oriented and should be stable.

Active transcript
  Redrawn by Ratatui every frame.
  It contains thinking, streaming text, running tools.

Submitted follow-up
  Currently persisted immediately, but hidden from
  native scrollback while active to avoid autoscroll.

Why It Got Weird

Each tactical fix pushed on one edge of this split and exposed another edge.

Native scrollback autoscrolls

If the submitted user line is printed immediately, the terminal may scroll so that line becomes the visible anchor.

Ratatui owns live text

Thinking and streaming are not printed into native scrollback. They are drawn in the inline viewport.

The viewport height changed

When active text appeared, the inline viewport grew from composer height to almost the full terminal.

The prompt was suppressed

Suppressing the prompt avoided autoscroll, but it also made the user message invisible during the delay.

The Target Model

This is the architecture I would move toward: a stable app-owned live turn, with native scrollback only used for committed output.

Native scrollback
Append-only committed transcript. No clear/replay for normal turns.
Pending user message
Shows instantly in Ratatui after Enter: > yo
Active assistant state
Shows waiting, thinking, streaming, running tools.
Composer
Stable height. No viewport expansion just because activity starts.
Better Pseudo HTML
<div id="terminal-transcript">
  <CommittedTranscript owner="native-scrollback" />

  <LiveTurn owner="ratatui">
    <PendingUserMessage text="yo" />
    <AssistantStatus state="waiting" />
  </LiveTurn>
</div>
